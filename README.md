# Лабораторная работа №9: Каталог игр (Jetpack Compose)

##  Описание проекта
Приложение представляет собой интерактивный каталог игр, разработанное на языке **Kotlin** с использованием фреймворка **Jetpack Compose**.
Основная цель работы — создание полноценного UI с управлением состоянием, обработкой событий и адаптивной версткой.

##  1. Low-fi Прототип
Перед началом разработки был создан низкоконтрастный прототип для планирования структуры экрана.


**Элементы макета:**
- **Изображение:** Постер игры (занимает верхнюю часть экрана).
- **Заголовок:** Название игры.
- **Описание:** Краткий сюжет или особенности игры.
- **Навигация:** Кнопки "Назад" и "Вперед" для переключения между элементами.
- **Индикатор:** Текущий номер элемента из общего количества (например, 2 / 5).

## 2. Реализация интерфейса (Static Layout)
Интерфейс собран с использованием следующих компонуемых функций (Composables):

| Элемент | Описание использования |
| :--- | :--- |
| `Column` | Основной вертикальный контейнер для расположения элементов друг под другом. |
| `Image` | Отображение постера игры с использованием `painterResource` и `ContentScale.Crop`. |
| `Text` | Вывод названия (`headlineMedium`) и описания (`bodyLarge`). |
| `Row` | Горизонтальное расположение кнопок навигации внизу экрана. |
| `Button` | Интерактивные элементы для смены состояния. |
| `Spacer` | Добавление отступов для визуального разделения блоков. |
| `Modifier` | Управление размерами (`fillMaxWidth`, `height`), отступами (`padding`) и выравниванием. |

##  3. Логика и Состояние (State & Interaction)
Для реализации интерактивности было использовано управление состоянием в Compose:

### Управление состоянием
- Использована конструкция `var currentIndex by remember { mutableStateOf(0) }`.
- Переменная `currentIndex` хранит индекс текущей отображаемой игры из списка.
- При изменении значения `currentIndex` происходит автоматическая перерисовка (Recomposition) экрана с новыми данными.

Реализована циклическая навигация по списку:
